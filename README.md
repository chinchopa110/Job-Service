# Job-Service

## Функциональный требования
### Два режима работы: работник и работодатель
#### Работник
- Создания и редактирование собственного профиля
- Просмотр всех вакансий с фильтрами(зп, город, удаленка, дата публикации, доступные профессии)
- Просмотр списка работодателей
- Просмотра всех вакансий от конкретного работодателя
- Возможность отклика на вакансию
### Работодатель
- Создание и редактирования профиля компании
- Публикация вакансии
- Удаление вакансии
- Просмотр откликов на вакансию

## API Endpoints

### Workers
- `POST /api/workers` - создать работника
- `PUT /api/workers` - обновить работника

### Employers
- `POST /api/employers` - создать работодателя
- `PUT /api/employers` - обновить работодателя
- `GET /api/employers` - получить всех работодателей

### Job Postings
- `POST /api/jobs` - создать вакансию
- `DELETE /api/jobs/{id}` - удалить вакансию
- `GET /api/jobs/search` - поиск вакансий
- `GET /api/jobs/employer/{id}` - вакансии работодателя

### Replies
- `POST /api/replies` - создать отклик
- `GET /api/replies/job/{id}` - отклики на вакансию


### Функциональность клиента

#### Режим «Работник»
- **Мой профиль** — создание профиля (имя, email, навыки, опыт, город, резюме)
- **Поиск вакансий** — фильтрация по городу, профессии, зарплате (мин/макс), удалённой работе; отклик на выбранную вакансию
- **Работодатели** — список всех работодателей; просмотр вакансий конкретного работодателя

#### Режим «Работодатель»
- **Профиль компании** — создание профиля (название, email, описание, город)
- **Управление вакансиями** — публикация вакансии, удаление вакансии, просмотр откликов
